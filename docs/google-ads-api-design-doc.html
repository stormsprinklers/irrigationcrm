Google Ads API Tool — Design Documentation

Company Name

Storm Sprinklers (product name: Irrigation CRM)

Business Model

Storm Sprinklers is an irrigation services company based in Utah County, Utah. We operate our own customer-facing website at www.stormsprinklers.com to generate leads for sprinkler repair, installation, seasonal blowouts, and related services.

We run paid advertising only for our own business through our own Google Ads manager account (MCC) and linked customer account. We advertise Google Search, Display, Performance Max, and Local Services Ads (LSA) campaigns that drive traffic to stormsprinklers.com and phone calls for our field team.

We do not manage Google Ads for any other companies or clients. Irrigation CRM is an internal operations platform used exclusively by Storm Sprinklers staff — not a SaaS product resold to third parties.

Tool Access / Use

Irrigation CRM is hosted at https://crm.stormsprinklers.com. The application requires staff login (company credentials via secure session authentication). There is no public sign-up and no external user access.

  • Who uses it: Storm Sprinklers owners, office staff, and marketing managers.
  • Primary use: View Google PPC and Local Services Ads performance alongside Meta ads in a unified Marketing → Ads dashboard.
  • Connection flow: An authenticated admin connects their Google account once in Settings → Integrations → Google Ads via OAuth 2.0 (adwords scope).
  • Data sharing: Google Ads data is displayed only inside the CRM to logged-in staff. We do not sell, reshare, or expose Google Ads data to unrelated third parties.
  • Planned management features: Pause or enable campaigns and adjust daily budgets from the CRM when seasonal demand changes (e.g., winter blowout season vs. spring startup).

A public marketing homepage at / describes the application for Google OAuth verification; all reporting features require staff authentication.

Tool Design

Architecture overview

Flow: Staff browser → Irrigation CRM (Next.js on Vercel) → PostgreSQL (OAuth refresh token, selected customer ID) → Google Ads API v18 (server-side, with developer token + OAuth access token) → Dashboard UI (Marketing → Ads)

Reporting (live today)

When a staff member opens Marketing → Ads or changes the date range, the CRM server calls the Google Ads API using Google Ads Query Language (GAQL) via GoogleAdsService.Search. Results are returned to the browser and rendered in the dashboard. Metrics include spend, impressions, clicks, CPC, conversions, conversion value, and ROAS aggregated by campaign.

Campaign rows include channel type (Search, Display, Performance Max, Local Services) so PPC and LSA campaigns appear in the appropriate tabs. The staff member can filter by preset ranges (7 / 30 / 90 days, month-to-date, year-to-date, all time, or custom dates).

Account linking (live today)

After OAuth connect, the admin selects which Google Ads customer account to report on from accessible accounts returned by CustomerService.ListAccessibleCustomers. Selection is stored per company in the CRM database.

Campaign management (planned — same tool)

Planned features use the same OAuth connection and developer token. Staff will be able to pause or enable campaigns and update campaign budgets from the Ads dashboard when business conditions change. Local Services Ads will show lead counts, cost per lead, and booked jobs in the dedicated LSA tab using LOCAL_SERVICES channel campaign data.

Security

  • OAuth refresh tokens stored server-side only (PostgreSQL); never sent to the browser.
  • Google Ads developer token stored as a server environment variable (GOOGLE_ADS_DEVELOPER_TOKEN).
  • All Google Ads API calls originate from CRM backend route handlers after session authentication.
  • Disconnect removes stored OAuth tokens immediately.

API Services Called

Status API service / method Purpose
Live CustomerService.ListAccessibleCustomers List Google Ads accounts the connected user can access; populate account picker in Settings.
Live GoogleAdsService.Search (GAQL on customer) Read customer descriptive name and manager flag for each accessible account.
Live GoogleAdsService.Search (GAQL on campaign, campaign_budget, metrics) Pull campaign performance reports: name, status, channel type, budget, cost, impressions, clicks, conversions, conversion value for the selected date range. Includes LOCAL_SERVICES channel for LSA campaigns.
Planned CampaignService.MutateCampaigns Pause or enable campaigns from the CRM when seasonality or capacity requires it.
Planned CampaignBudgetService.MutateCampaignBudgets Adjust daily campaign budgets for PPC and LSA from the Budgets tab.
Planned GoogleAdsService.Search (GAQL on local services lead resources) Display LSA lead volume, CPL, and booking metrics in the Google LSA tab.

OAuth scope: https://www.googleapis.com/auth/adwords

API version: Google Ads API v18 (REST endpoint googleads.googleapis.com/v18)

Data Handling Summary

  • Single-tenant internal tool for Storm Sprinklers only.
  • Google Ads data used solely for internal marketing decisions and operational reporting.
  • No resale, no multi-client ad management, no sharing with unrelated third parties.
  • Users can disconnect Google Ads at any time; stored OAuth tokens are deleted on disconnect.
  • Privacy policy: https://crm.stormsprinklers.com/privacy
